使用指引与常见问题

Obsidian 仓库名

填写你要把 Markdown 文件保存到的仓库名字。仓库名可以在 Obsidian 页面左下角看到；如果你有多个仓库，请填写你希望保存到的那个仓库名。

务必填写正确：如果仓库名填错，Obsidian 会弹出 "repository not found" 等错误，但本扩展无法感知 Obsidian 内部是否保存成功，状态栏仍会显示「已保存到 Obsidian」。请确保与 Obsidian 左下角显示的名称完全一致（区分大小写）。

默认保存文件夹

Markdown 文件默认保存到仓库根目录的指定文件夹里。不填写则直接保存到根目录中。

日期子目录模板

支持 {{YYYY}}/{{MM}}、{{YYYY}}/doc、{{MM}}/{{DD}}/inbox 这类带日期、时间以及固定文件夹的命名模式，填写后会保存到对应的日期子目录中。不填写则保存到「默认保存文件夹」中。

可用变量：{{YYYY}} 年、{{MM}} 月、{{DD}} 日、{{HH}} 时、{{mm}} 分、{{ss}} 秒。

选中文字自动带入正文

如果在页面上用鼠标框选了文字，打开本插件时会将选中的文字自动粘贴到文本框中。关闭此选项则不带入正文。

保留框选内容中的图片链接

开启后，如果框选的内容中包含图片，图片链接会以 Markdown 图片语法一并带入正文。关闭则只保留纯文本。

呼出时捕获视频播放进度

开启后，如果页面正在播放视频，打开插件时会自动把当前播放进度以可点击链接的形式插入草稿。支持 YouTube、Bilibili、Vimeo、Dailymotion、Twitch 等平台；其他平台会回退到当前页面链接。

为什么有些图片框选了但没有被保留？

本功能只能提取标准 HTML 图片标签（<img>）。如果网站使用 Shadow DOM、Canvas 或其他特殊渲染方式（如部分视频网站的评论区），图片可能无法被自动提取，建议遇到这类情况时手动截图。

Frontmatter

勾选对应的选项后，保存的 Markdown 文件会在开头生成 YAML frontmatter，包含对应的内容。例如勾选 title 会写入页面标题，勾选 url 会写入来源链接。

tags

对应 frontmatter 里的 tags，使用逗号分隔多个标签。请注意标签本身不支持空格。

为什么 popup 里有时看不到 tags 栏？

tags 栏是否显示由 Frontmatter 里的 tags 开关控制。只有在 popup 中勾选 tags，或者在设置页默认开启 tags 时，标签栏才会出现。

临时 tag 和全局 tag 有什么区别？

在 popup 里点击「+ 添加」新增加的 tag 会显示为浅绿底、虚线边框，表示它只在当前这条笔记生效，不会同步到设置页的常用标签池。来自设置页常用标签池的 tag 显示为白底实线，是全局 tag。

保存时浏览器询问是否打开 Obsidian？

这是 Chrome 的安全机制。首次在某个网站保存时，浏览器会询问「是否允许此网站打开 Obsidian」，请勾选始终允许，以后在该网站保存就不会再弹窗。

界面语言

扩展根据 Chrome 的界面语言自动切换中文或英文。如需更换语言，请调整 Chrome 设置中的显示语言。

为什么 Obsidian 没有打开或笔记没保存？
  • 确认「Obsidian 仓库名」与 Obsidian 左下角显示的名称完全一致（区分大小写）。
  • 确认「默认保存文件夹」和「日期子目录模板」拼写正确，且该文件夹在仓库中真实存在。
  • 确认 Obsidian 已安装且能处理 obsidian:// 链接。
